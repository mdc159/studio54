## What SecureReview AI Should Never Claim

### Prohibited Claims

#### 1. Automated Security Decisions
- **Never claim:** "AI makes final security decisions" or "replaces human security judgment"
- **Why:** Regulatory environments require human accountability and validation for security decisions
- **Instead say:** "AI provides enhanced analysis to support human security decision-making and validation processes"

#### 2. Guaranteed Regulatory Approval or Universal Compliance
- **Never claim:** "Guaranteed regulatory approval" or "meets all compliance requirements"
- **Why:** Each organization has specific regulatory frameworks and approval processes
- **Instead say:** "Designed to support customer regulatory approval processes with specialized compliance documentation and professional services"

#### 3. Performance Equivalence to Cloud Solutions
- **Never claim:** "Equivalent performance to cloud AI" or "no performance trade-offs"
- **Why:** On-premise deployment and regulatory constraints create inherent performance limitations
- **Instead say:** "Performance optimized for regulatory-constrained environments with measurable improvement over current static analysis tools"

#### 4. Complete Data Sovereignty Claims
- **Never claim:** "Complete data sovereignty" or "air-gapped deployment"
- **Why:** These create legal/compliance risks and depend on customer-specific implementation
- **Instead say:** "Designed for deployment within customer-controlled computing environments per regulatory requirements"

*Combined Version A's automation and compliance prohibitions with Version B's sovereignty and performance prohibitions. All four are necessary to prevent legal/technical/sales problems.*
