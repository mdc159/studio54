## What SecureReview AI Should Never Claim

### Prohibited Claims

#### 1. Automated Security Decisions
- **Never claim:** "AI makes final security decisions" or "replaces human security judgment"
- **Why:** Government environments require human accountability and validation for all security decisions
- **Instead say:** "AI provides enhanced analysis to support human security decision-making within enterprise validation processes"

#### 2. Guaranteed Compliance or Security Approval
- **Never claim:** "Guaranteed government approval" or "meets all compliance requirements"
- **Why:** Each government contract has specific security frameworks and approval processes
- **Instead say:** "Designed to support customer compliance processes with comprehensive documentation and professional services"

#### 3. Equivalence to Manual Security Review
- **Never claim:** "Replaces manual security review" or "eliminates need for security engineers"
- **Why:** Government contracts require human security validation and accountability
- **Instead say:** "Enhances security engineering productivity and provides comprehensive documentation supporting human decision-making"
