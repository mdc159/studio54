## What SecureReview AI Should Never Claim

### Prohibited Claims

#### 1. Automated Security Decisions
- **Never claim:** "AI makes final security decisions" or "replaces human security judgment"
- **Why:** Government environments require human accountability and validation for all security decisions and compliance documentation
- **Instead say:** "AI provides enhanced analysis to support human security decision-making within established enterprise validation processes"

#### 2. Guaranteed Compliance or Security Approval
- **Never claim:** "Guaranteed government approval" or "meets all compliance requirements automatically"
- **Why:** Each government contract has specific security frameworks and approval processes requiring human validation
- **Instead say:** "Designed to support customer compliance processes with comprehensive documentation and professional services experienced with government requirements"

#### 3. Complete Replacement of Security Engineering
- **Never claim:** "Replaces manual security review" or "eliminates need for security engineers"
- **Why:** Government contracts require human security validation, accountability, and compliance documentation
- **Instead say:** "Enhances security engineering productivity and provides comprehensive documentation supporting human decision-making and compliance requirements"

#### 4. Universal Deployment Capability
- **Never claim:** "Works in any environment" or "no deployment constraints"
- **Why:** Both government cloud and air-gapped deployments have specific requirements and limitations
- **Instead say:** "Flexible deployment model designed for government cloud and air-gapped environments with appropriate professional services support"

*Combined Version A's automation and compliance prohibitions with Version B's human validation requirements and Version A's deployment reality acknowledgment.*
