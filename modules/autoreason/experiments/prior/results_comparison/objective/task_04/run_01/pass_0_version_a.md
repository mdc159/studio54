## WHAT WE SHOULD NEVER CLAIM

### **Prohibited Positioning Statements**

**❌ Never Say:**
- "Better AI than OpenAI/GitHub" - *Avoid direct AI capability comparisons*
- "100% accurate bug detection" - *AI is probabilistic, not deterministic*
- "Replaces human code reviewers" - *Positions as job threat*
- "Works with any infrastructure" - *We have specific requirements*
- "Instant deployment" - *Enterprise deployment takes time*
- "No learning curve" - *There's always some adoption period*
- "Cheapest solution" - *We're premium-priced for security*

**❌ Avoid These Comparisons:**
- Direct speed/accuracy claims vs. cloud providers without context
- "Military-grade security" (overused and meaningless)
- "Revolutionary" or "breakthrough" technology claims
- ROI guarantees or specific percentage improvements without caveats

**✅ Always Include Caveats:**
- Model performance depends on training data quality
- Deployment timeline varies by infrastructure complexity  
- Results based on customer-specific implementations
- Continuous monitoring required for optimal performance
