**MEMORANDUM**

TO: All Employees
FROM: Executive Team
DATE: [Date]
RE: Artificial Intelligence Tool Usage Policy

## SCOPE

1. This policy applies to all 200 employees across engineering, sales, and other departments.
2. Policy covers all AI-powered tools used for work purposes, including but not limited to code generation, content creation, and data analysis tools.
3. Violations will be tracked through existing incident management and HR disciplinary processes.

## PERMITTED USES

1. GitHub Copilot Business is approved for code generation and completion for the 80 licensed seats.
2. AI tools may be used for internal documentation, meeting summaries, and non-customer-facing content creation.
3. Sales teams may use AI for drafting initial email templates and proposal outlines for internal review before customer communication.
4. AI-generated code must pass existing code review processes before deployment.
5. All AI tool usage must comply with our SOC2 Type II, GDPR, and pending FedRAMP requirements.

## PROHIBITED USES

1. Inputting customer PII, financial data, or any data covered by Data Processing Agreements into AI tools (motivated by GDPR obligations and DPA violations identified by legal counsel).
2. Pasting customer database schemas, API keys, or proprietary system architectures into AI services (motivated by incident #1 - engineer database schema exposure).
3. Using AI-generated content containing third-party copyrighted material without verification and attribution (motivated by incident #2 - sales rep copyright violation).
4. Committing AI-generated code without removing training data artifacts, including license headers from other projects (motivated by incident #3 - GPL license contamination).
5. Using AI tools other than approved GitHub Copilot Business without prior IT and Legal approval.
6. Sharing AI-generated code externally without disclosure that portions may not be copyrightable (based on legal counsel guidance).

## ENFORCEMENT

1. Engineering managers will verify AI usage compliance during existing weekly code reviews.
2. Sales managers will review AI-generated customer communications before sending, using current approval workflows.
3. IT will monitor and block unauthorized AI tool access through existing network controls and software management.
4. Violations will result in progressive discipline through existing HR processes: first offense (written warning), second offense (performance improvement plan), third offense (termination consideration).
5. All AI tool incidents must be reported to IT Security within 24 hours using current incident reporting procedures.
6. Quarterly compliance reviews will be conducted by Legal and IT teams using existing audit processes.

This policy is effective immediately. Questions should be directed to your manager or IT Security.
