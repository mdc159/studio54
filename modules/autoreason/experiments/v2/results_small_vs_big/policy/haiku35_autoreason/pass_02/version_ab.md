MEMO: AI Tool Usage Policy
Effective Date: [Current Date]

Scope:
1. This policy applies to all 200 employees of our B2B SaaS company handling sensitive financial and personal data.
2. The policy covers AI tools used across engineering (73% current usage), sales (45% current usage), and other departments.
3. Applies to currently licensed tools: GitHub Copilot Business and Slack (with AI features disabled).
4. This policy supersedes all previous informal AI tool practices.

Permitted Uses:
1. GitHub Copilot Business is the only approved AI coding assistant for engineering teams.
2. AI tools may be used for:
   a. Code suggestion and autocompletion
   b. General writing assistance
   c. Task brainstorming and ideation
3. Users must review and validate all AI-generated content before use.
4. AI tool usage must comply with:
   a. SOC2 Type II certification requirements
   b. Pending FedRAMP authorization standards
   c. Existing Data Processing Agreement (DPA) terms
5. Employees must ensure AI-generated content does not compromise customer data protection agreements.

Prohibited Uses:
1. Inputting customer Personal Identifiable Information (PII) or financial data into any AI tool. (Ref: Incident 1, GDPR obligations)
2. Using AI-generated content that includes:
   a. Verbatim copyrighted material from external sources
   b. Competitor's marketing language
   (Ref: Incident 2, Legal counsel guidance)
3. Committing AI-generated code that:
   a. Includes third-party license headers
   b. May not be independently copyrightable
   (Ref: Incident 3, Legal counsel finding on code copyrightability)
4. Using AI tools not explicitly approved by IT and Legal departments.
5. Bypassing company data protection agreements through personal AI tool accounts.
6. Generating client-facing materials without complete human review.

Enforcement:
1. Violations will be addressed through existing performance management processes.
2. First offense: Formal written warning using current HR documentation systems.
3. Repeated violations may result in:
   a. Revocation of AI tool access through existing system permissions
   b. Disciplinary action up to termination
4. Managers will use existing performance review mechanisms to ensure compliance.
5. Any policy violations must be immediately reported to management.
