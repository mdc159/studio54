MEMO: AI Tool Usage Policy
Effective Date: [Current Date]

Scope:
1. This policy applies to all 200 employees of [Company Name].
2. Covers AI tool usage across engineering, sales, and other departments within the $50K annual AI tooling budget.
3. Applies to currently licensed tools: GitHub Copilot Business (80 seats) and Slack (with AI features disabled).
4. This policy supersedes all previous informal AI tool practices.

Permitted Uses:
1. GitHub Copilot Business is the only approved AI coding assistant for engineering teams (80 seat allocation).
2. AI tools may be used for:
   a. Code suggestion and autocompletion
   b. General writing assistance
   c. Task brainstorming and ideation
3. Users must complete a mandatory "AI-generated Content Review" checklist before using any AI-generated content.
4. AI tool usage must comply with SOC2, GDPR, and pending FedRAMP authorization requirements.

Prohibited Uses:
1. Inputting any customer data into external AI tools, specifically:
   a. Exposing database schemas (Ref: Incident 1 - database schema exposure)
   b. Sharing Personal Identifiable Information (PII) 
   c. Violating existing Data Processing Agreements (DPAs)
2. Using AI-generated content that includes:
   a. Verbatim copyrighted material from external sources
   b. Competitor's marketing language (Ref: Incident 2 - copyrighted marketing copy)
3. Committing AI-generated code that:
   a. Includes third-party license headers
   b. May not be independently copyrightable (Ref: Incident 3 - GPL license header)
4. Using AI tools not explicitly approved within the $50K annual tooling budget.
5. Bypassing company data protection agreements through personal AI tool accounts.

Enforcement:
1. Tracking AI tool usage:
   a. Engineering teams must log AI tool usage through existing project management tools
   b. Quarterly audit of AI tool usage against 73% engineering and 45% sales baseline
2. Violations addressed through existing performance management processes:
   a. First offense: Formal written warning
   b. Repeated violations result in:
      - Revocation of AI tool access via existing system permissions
      - Disciplinary action up to termination
3. Mandatory quarterly training on AI tool usage and data protection
4. Department heads responsible for ensuring team compliance with policy
5. Any policy violations must be immediately reported to management and IT security
