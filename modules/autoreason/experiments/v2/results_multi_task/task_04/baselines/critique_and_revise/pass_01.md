## Messaging Guidelines & Boundaries

### ✅ **Approved Messaging:**

**Security & Compliance:**
- "Eliminates data sovereignty concerns associated with cloud AI tools"
- "Designed to meet stringent regulatory compliance requirements"
- "Maintains complete audit trails for compliance reporting"
- "Reduces security risks compared to cloud-based alternatives"

**Capabilities & Performance:**
- "Delivers enterprise-grade AI code review capabilities"
- "Significantly improves review efficiency compared to manual processes"
- "Provides consistent security vulnerability detection"
- "Customizable to organizational coding standards and security policies"

**Business Value:**
- "Purpose-built for security-conscious enterprises"
- "Enables AI adoption within existing security frameworks"
- "Scales with organizational growth and requirements"

### ❌ **Prohibited Claims:**

**Absolute Security Claims:**
- Never claim "100% secure" or "unhackable"
- Avoid "guarantees" regarding security outcomes
- Don't claim immunity from all security risks

**Performance Absolutes:**
- Never claim "faster than all cloud solutions"
- Avoid "best AI models available"
- Don't claim "zero maintenance required"

**Market Position Overclaims:**
- Avoid "only solution" unless specifically true and verifiable
- Don't claim superiority in all use cases
- Never disparage cloud solutions as "insecure" - focus on fit for purpose

**Capability Overpromises:**
- Don't claim to replace all human reviewers
- Avoid "catches all vulnerabilities"
- Never promise specific ROI percentages without qualification

### ⚠️ **Required Disclaimers:**
- "Results may vary based on deployment configuration and usage patterns"
- "Compliance with specific regulations should be verified with legal counsel"
- "Performance comparisons based on internal testing and customer reports"

## Sales Enablement Framework

### Discovery Questions by Persona

**For CISOs:**
1. "What compliance frameworks does your organization operate under?"
2. "How does your security team currently evaluate AI tools for development use?"
3. "What would be the impact of a code exposure incident on your organization?"
4. "Do you have data residency requirements that affect tool selection?"
5. "How do you balance innovation enablement with security requirements?"

**For Engineering Leadership:**
1. "What percentage of development time is currently spent on code reviews?"
2. "What are your biggest code quality and security challenges?"
3. "How do current review processes impact release velocity?"
4. "What AI tools have your developers requested that you've had to decline?"
5. "How do you measure and improve code review effectiveness?"

**For Development Teams:**
1. "What aspects of code review are most time-consuming for your team?"
2. "What types of issues do manual reviews frequently miss?"
3. "How familiar is your team with AI-powered development tools?"
4. "What would improve your code review experience most?"

### Qualification Criteria

**High-Fit Indicators:**
- Operates under strict regulatory compliance (SOX, HIPAA, GDPR, FedRAMP)
- Has existing on-premise infrastructure capabilities
- Currently restricts or prohibits cloud-based AI tools
- Development team size 20+ developers
- Has experienced security incidents or compliance challenges
- Values intellectual property protection highly

**Medium-Fit Indicators:**
- General security consciousness without specific regulatory requirements
- Hybrid cloud/on-premise infrastructure
- Some cloud AI tool usage with security concerns
- Smaller development teams (5-20 developers)

**Low-Fit Indicators:**
- Cloud-first infrastructure strategy
- No regulatory compliance requirements
- Comfortable with existing cloud AI tool usage
- Very small development teams (<5 developers)
- Cost-sensitive with limited security budget
