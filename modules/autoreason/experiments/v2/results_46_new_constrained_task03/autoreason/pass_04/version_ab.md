# AI Tool Usage Policy
**Effective Date:** [Date] | **Owner:** Engineering & Legal | **Applies To:** All Employees

---

## Scope

1. This policy governs all AI tools—code assistants, text generators, and chat interfaces—used by all 200 employees across all functions.
2. This policy applies on company devices, personal devices used for work, and all third-party AI services, whether company-licensed or personal accounts.
3. All informal AI tool use not explicitly permitted below must cease on the effective date.

---

## Permitted Uses

1. Engineers with an assigned GitHub Copilot Business seat (80 of 120 seats licensed) may use it for code completion, generation, and review in approved repositories. Engineering leadership manages seat assignment.
2. No AI tool is approved for sales, other non-engineering functions, or the 50 employees in other roles. Any use by these groups is prohibited under Prohibited Uses item 4.
3. All AI-generated code submitted via pull request goes through the existing PR approval process. Reviewers flag any license headers, license-referencing comments, or undisclosed AI-generated code for Legal review before merge.

---

## Prohibited Uses

1. **No customer data in third-party AI tools.** Inputting PII, financial data, or any customer-identifiable information into any AI service outside company-controlled systems is prohibited. *[Basis: Incident 1—engineer pasted customer database schema into ChatGPT; outside counsel confirmed this likely violates existing DPA terms; SOC2 Type II, GDPR, and pending FedRAMP authorization require data handling controls.]*
2. **No AI-generated content transmitted externally without prior written manager approval.** Before any AI-generated text is sent to customers, prospects, or partners, the sender must obtain email approval from their direct manager. That email constitutes the record of approval and is retained under existing email retention controls. *[Basis: Incident 2—sales rep transmitted verbatim competitor copyrighted marketing copy generated by AI.]*
3. **No AI-generated code merged without PR review clearance.** Engineers disclose AI-generated code in pull requests; no such code is merged until reviewers confirm no license headers or undisclosed AI-generated content are present, per the existing PR approval process. *[Basis: Incident 3—intern committed GPL-licensed AI-generated code; outside counsel confirmed AI-generated code may not be copyrightable.]*
4. **No unapproved AI tools.** Engineers without an assigned GitHub Copilot Business seat, and all non-engineering employees, use no AI tool for work purposes. *[Basis: 80 seats licensed for 120 engineers; no other AI tool has completed DPA and IP review; outside counsel confirmed third-party AI input of customer data likely violates DPA terms.]*
5. **No enabling of Slack AI features.** Slack AI features are disabled at the administrative level. Employees must not submit requests to IT or to Slack to enable them. *[Basis: Slack AI features are currently disabled; Slack processes internal communications containing customer and financial data subject to SOC2, GDPR, and FedRAMP controls; outside counsel confirmed inputting customer data into third-party AI services likely violates DPA terms.]*

---

## Enforcement

1. All violations are reported to the employee's direct manager and Legal the same business day via existing SOC2 incident reporting channels. Violations involving customer PII, financial data, or FedRAMP-relevant systems are additionally escalated to the CEO the same business day.
2. Confirmed violations follow the existing disciplinary process: documented warning, performance plan, or termination.
3. Managers are accountable for team compliance. Quarterly access reviews required for SOC2 include verification of AI tool usage against this policy.
