# AI Tool Usage Policy
**Effective Date:** [Date] | **Owner:** Engineering & Legal | **Applies To:** All Employees

---

## Scope

1. This policy governs all use of AI tools—including code assistants, text generators, and chat interfaces—by all 200 employees across engineering, sales, and all other functions.
2. This policy applies to AI use on company devices, personal devices used for work, and any third-party AI service, whether company-licensed or personal.
3. This policy supersedes all informal practices currently in use.

---

## Permitted Uses

1. Engineers may use **GitHub Copilot Business** (80 licensed seats) for code completion, generation, and review within approved repositories. Seat allocation is managed by Engineering leadership.
2. Sales and other functions may use AI tools for internal drafting (outlines, first drafts) where output is reviewed and rewritten by a human before any external communication is sent.
3. All AI-generated code merged into production must include a comment identifying it as AI-assisted, enabling review under existing pull request approval processes.
4. Questions about tool eligibility are directed to Legal before first use. The $50K annual AI tooling budget is the ceiling for any new tool approvals; requests go through Finance and Legal.

---

## Prohibited Uses

1. **No customer data in third-party AI tools.** Inputting PII, financial data, or any customer-identifiable information into any AI service outside company-controlled systems is prohibited. *[Basis: Incident 1—engineer pasted customer database schema into ChatGPT; outside counsel confirmed this likely violates existing DPA terms; SOC2 Type II, GDPR, and pending FedRAMP authorization require data handling controls.]*
2. **No unreviewed AI-generated external content.** AI-generated text sent externally—to customers, prospects, or partners—must be manually reviewed and rewritten to confirm it contains no third-party copyrighted material. *[Basis: Incident 2—sales rep transmitted verbatim competitor copyrighted marketing copy generated by AI.]*
3. **No AI-generated code committed without license review.** Engineers must run all AI-generated code through the existing legal review checklist before commit if the code includes any license header, comment referencing a license, or is flagged by Copilot's license filter. *[Basis: Incident 3—intern committed GPL-licensed AI-generated code; outside counsel flagged that AI-generated code may not be copyrightable, creating IP exposure.]*
4. **No AI features in company Slack.** AI features in Slack remain disabled. Employees do not request or enable them. *[Basis: Current approved configuration; enabling them would create uncontrolled data exposure channels.]*
5. **No use of non-approved AI tools for work tasks** until Legal completes a DPA and IP review and written approval is issued.

---

## Enforcement

1. Violations are reported to the employee's direct manager and the Legal team within 24 hours of discovery.
2. Engineering violations are detectable through existing pull request history, commit logs, and access logs—no new tooling is required.
3. Confirmed violations follow the existing disciplinary process: documented warning, performance plan, or termination, depending on severity and recurrence.
4. Managers are accountable for their team's compliance. Quarterly access reviews—already required for SOC2—include verification of AI tool usage against this policy.
