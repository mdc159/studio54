# AI Tool Usage Policy
**Effective Date:** [Date] | **Owner:** Engineering & Legal | **Applies To:** All Employees

---

## Scope

1. This policy governs all use of AI tools—including code assistants, text generators, and chat interfaces—by all 200 employees across engineering, sales, and all other functions.
2. This policy applies to AI use on company devices, personal devices used for work, and any third-party AI service, whether company-licensed or personal.

---

## Permitted Uses

1. Engineers may use **GitHub Copilot Business** (80 licensed seats) for code completion, generation, and review within approved repositories. Seat allocation is managed by Engineering leadership.
2. Sales and other functions may use AI tools for internal drafts only. AI-generated text is not sent externally until a human has rewritten it and confirmed it contains no verbatim third-party copyrighted material.
3. All AI-generated code submitted via pull request is subject to existing PR approval processes. Reviewers flag any license headers or license-referencing comments for legal review before merge.
4. New AI tool requests are submitted to Legal for DPA and IP review before first use. Approved tools are funded within the $50K annual AI tooling budget; requests go through Finance and Legal.

---

## Prohibited Uses

1. **No customer data in third-party AI tools.** Inputting PII, financial data, or any customer-identifiable information into any AI service outside company-controlled systems is prohibited. *[Basis: Incident 1—engineer pasted customer database schema into ChatGPT; outside counsel confirmed this likely violates existing DPA terms; SOC2 Type II, GDPR, and pending FedRAMP authorization require data handling controls.]*
2. **No unreviewed AI-generated external content.** AI-generated text is not sent to customers, prospects, or partners without manual rewriting that confirms the absence of verbatim third-party copyrighted material. *[Basis: Incident 2—sales rep transmitted verbatim competitor copyrighted marketing copy generated by AI.]*
3. **No AI-generated code committed without license review.** Engineers submit all AI-generated code through the existing legal review checklist before commit if the code includes any license header or license-referencing comment. *[Basis: Incident 3—intern committed GPL-licensed AI-generated code; outside counsel flagged that AI-generated code may not be copyrightable.]*
4. **No AI features in company Slack.** Slack AI features remain disabled. Employees do not request or enable them. *[Basis: Company Slack is an approved tool with AI features currently disabled.]*
5. **No non-approved AI tools for work tasks** until Legal issues written approval following DPA and IP review.

---

## Enforcement

1. Violations are reported to the employee's direct manager and to Legal by the discoverer on the same business day, using existing email and ticketing channels already required for SOC2 incident reporting.
2. Confirmed violations follow the existing disciplinary process: documented warning, performance plan, or termination, depending on severity and recurrence.
3. Managers are accountable for their team's compliance. Quarterly access reviews—already required for SOC2—include verification of AI tool usage against this policy.
