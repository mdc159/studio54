# AI Tool Usage Policy
**Effective Date:** [Date] | **Owner:** Engineering & Legal | **Applies To:** All Employees

---

## Scope

1. This policy governs all AI tools—code assistants, text generators, and chat interfaces—used by all 200 employees in all functions.
2. This policy applies on company devices, personal devices used for work, and all third-party AI services, whether company-licensed or personal.

---

## Permitted Uses

1. Engineers with an assigned GitHub Copilot Business seat (80 of 120 seats licensed) may use it for code completion, generation, and review in approved repositories. Engineering leadership manages seat assignment.
2. No AI tool is approved for sales or other non-engineering functions. Requests are held pending Legal's DPA and IP review, funded within the $50K annual budget, before any use begins.
3. All AI-generated code submitted via pull request goes through the existing PR approval process. Reviewers flag any license headers, license-referencing comments, or undisclosed AI-generated code for Legal review before merge.

---

## Prohibited Uses

1. **No customer data in third-party AI tools.** Inputting PII, financial data, or any customer-identifiable information into any AI service outside company-controlled systems is prohibited. *[Basis: Incident 1—engineer pasted customer database schema into ChatGPT; outside counsel confirmed this likely violates existing DPA terms; SOC2 Type II, GDPR, and pending FedRAMP authorization require data handling controls.]*
2. **No AI-generated content sent externally without manager written approval.** AI-generated text is not transmitted to customers, prospects, or partners unless the sender's direct manager approves the communication in writing, confirming no verbatim third-party copyrighted material is present. *[Basis: Incident 2—sales rep transmitted verbatim competitor copyrighted marketing copy generated by AI.]*
3. **No AI-generated code merged without PR review clearance.** Engineers disclose AI-generated code in pull requests; no such code is merged until reviewers confirm no license headers or undisclosed AI-generated content is present, per the existing PR approval process. *[Basis: Incident 3—intern committed GPL-licensed AI-generated code; outside counsel confirmed AI-generated code may not be copyrightable.]*
4. **No AI coding tools for unseated engineers.** Engineers without an assigned GitHub Copilot Business seat use no AI coding tool until a seat is assigned or Legal issues written approval for an alternative. *[Basis: 80 seats are licensed for 120 engineers; no other AI coding tool has completed DPA and IP review.]*
5. **No Slack AI features.** Employees do not request or enable Slack AI features. *[Basis: Slack AI features are currently disabled; Slack processes internal communications containing customer and financial data subject to SOC2, GDPR, and FedRAMP controls; outside counsel confirmed inputting customer data into third-party AI services likely violates DPA terms.]*

---

## Enforcement

1. All violations are reported to the employee's direct manager and Legal on the same business day via existing SOC2 incident reporting channels; violations involving customer PII, financial data, or FedRAMP-relevant systems are additionally escalated to the CEO the same business day.
2. Confirmed violations follow the existing disciplinary process: documented warning, performance plan, or termination.
3. Managers are accountable for team compliance. Quarterly access reviews required for SOC2 include verification of AI tool usage against this policy.
